# Senario 3: Membina CRUD Page

## Tujuan Senario

Senario ini digunakan apabila user ingin membina page yang boleh mengurus data.

CRUD bermaksud:

- **Create**: tambah rekod baharu
- **Read**: lihat rekod
- **Update**: kemas kini rekod
- **Delete**: padam rekod

Page Studio boleh digunakan untuk membina paparan CRUD, tetapi proses backend masih perlu disambungkan melalui Business Logic / API.

## Langkah-langkah

1. Buka **Page Studio**.
2. Create page baharu atau pilih page sedia ada.
3. Tambah field input yang diperlukan.
4. Tambah button yang sesuai seperti:
   - Save
   - Update
   - Delete
   - Cancel
   - Back
5. Susun form layout dengan kemas.
6. Tetapkan field yang wajib diisi.
7. Semak field name dan mapping data.
8. Sambungkan page dengan backend logic atau API jika diperlukan.
9. Klik **Save**.
10. Klik **Preview** untuk semak paparan.
11. Uji proses tambah, lihat, kemas kini dan padam rekod menggunakan sample data.

## Hasil Yang Dijangka

Page CRUD boleh digunakan untuk tambah, lihat, edit dan padam rekod berdasarkan konfigurasi backend.

## Nota Penting

- Page Studio menyediakan paparan frontend sahaja.
- Proses simpan, update dan delete data memerlukan backend logic.
- Permission user perlu disemak sebelum page diberikan kepada user sebenar.
- Uji page menggunakan sample data sebelum release.
